## 1. 比赛一句话简介

KDD Cup 2026 的主题是 **"用于复杂数据分析的数据智能体（Data Agents）"**。参赛队伍需要构建一个 **自主 AI Agent**：在收到用户的自然语言分析问题后，能够自行 **拆解任务 → 规划步骤 → 调用工具 → 在异构/多模态数据上进行多步推理 → 合成结构化答案**。最终提交形式是一个 Docker 镜像，由组委会在统一硬件、统一 LLM 的隔离环境中自动评测。

## 3. 任务设定

### 3.1 数据集：DataAgent-Bench

官方构造的基准测试集，每个 task 都是一个 **自包含的分析任务**，输入包括：

- **结构化数据**：JSON、CSV、关系型数据库
- **非结构化数据**：知识文档（短文本 ~ 128K+ token 的长文）
- **多模态数据**：Phase 2 引入 **图像型数据**（data images）和 **视频**

任务按难度分为 **Easy / Medium / Hard / Extreme** 四档。

### 3.2 两个阶段（Phase）

| 阶段 | 时间（AoE 时区） | 说明 |
|------|------------------|------|
| **Phase 1**：公共主赛道 | 2026-04-24 ~ 2026-05-23 | 全部注册队伍在公开 Leaderboard 上自动评测 |
| **Phase 2**：两个子赛道 | 2026-05-28 ~ 2026-06-30 | 更难的数据 + 新模态（图像、视频） |

**Phase 2 下的两条子赛道**：

1. **Leaderboard 子赛道（标准排行榜）**：继续按评测指标打分排名。
2. **Creative 子赛道（创意赛道）**：不看排行榜分数，而是考察 **系统设计、交互体验、决策过程的透明性、易用性**，由赞助商+组委会联合评审。

> 对于首次参赛的学生队伍，建议先聚焦 Phase 1，然后根据成绩决定是冲 Phase 2 的 Leaderboard 还是参加 Creative 子赛道。

## 6. 提交与评测规则（来自 rules 页面）

### 6.1 提交形式

- **唯一提交物：Docker 镜像**
- 命名：镜像 tag 为 `<team_id>:v<N>`，打包文件为 `<team_id>_v<N>.tar.gz`
- 提交方式：邮件提交 + Google Drive 分享链接
- **存档文件上限 10 GB**
- 提交频率：每天最多 1 次，Phase 1 总次数上限 30 次
- 本次评测未结束前，不能提交下一次

### 6.2 运行时规范

Agent 必须遍历 `/input` 目录下的所有任务目录，逐个处理，并把结果写入 `/output/task_<id>/prediction.csv`（UTF-8 编码标准 CSV）。

### 6.3 硬件与资源限制

| 项 | 限制 |
|----|------|
| CPU | 16 核（x86-64） |
| 内存 | 64 GB |
| GPU | **无 GPU** |
| 总运行时长 | 全部任务合计不超过 12 小时 |

### 6.4 LLM 使用规则（非常关键）

- **开发阶段**：本地调试可以随意接入任何 LLM。
- **官方评测阶段**：所有队伍 **必须统一使用组委会部署的 LLM**——页面写明型号为 `Qwen3.5-35B-A3B`，通过环境变量 `MODEL_API_URL`、`MODEL_API_KEY`、`MODEL_NAME` 注入。
- **严禁**：在代码里硬编码任何 API Key 或 LLM 服务地址；在评测环境里把别的 LLM 当作主推理引擎使用。

**关于 Qwen3.5-35B-A3B（查 HF 官方卡得到的关键信息，见 [方案调研综述](方案调研综述.md) §附录 A）**：

- 架构：MoE，总 35B / 激活 3B，256 专家；40 层；原生 256K 上下文（YaRN 可扩 1M）。
- **原生多模态**（Image-Text-to-Text + 视频理解），Phase 2 图像/视频可直接喂给它。
- 默认启用 thinking 模式（输出 `<think>...</think>`），占输出 token，注意在简单子任务里显式关闭。
- 百炼 API 价格：**输入 ¥0.4 / M tokens，输出 ¥3.2 / M tokens**（≤128K 上下文档），开发期全程走 API，Phase 1 全程费用 ¥100 量级。

### 6.5 评分方式（Leaderboard 赛道）

答案以"预测列 vs. 金标准列"的方式比对：

- 把每一列看成一个"值向量（multiset）"，**忽略列名，忽略行顺序**。
- Phase 1 规则（主页说明）：预测必须 **完整且准确地命中所有金标准列** 才能得到 1 分；缺一列或有一列不匹配直接 0 分；多预测的列不扣分。
- 规则页面给出更精细的公式：`Score = Recall - λ · (Extra Columns / Predicted Columns)`。
- 值归一化：数字保留 2 位小数；空值统一为空字符串；日期统一成 ISO 8601 (`YYYY-MM-DD`)。

### 6.6 Creative 子赛道评分

由赞助商与组委会联合评审，主要考察：**交互设计、决策过程透明性、创新性、易用性**。页面没有给出具体权重。

### 6.7 明令禁止的行为

1. 尝试访问外网，或绕过 `MODEL_API_URL` 去调用其他 LLM
2. 使用非 Qwen LLM 作为主推理引擎
3. 基础设施探测、攻击、容器逃逸
4. 修改 `/input`，或破坏注入的环境变量
5. 跨队共享 Docker 镜像
6. 泄露测试集答案、运行时人为干预

## 9. 组队与分工建议（3 学生 + 1 Advisor）

本队由 3 位学生作为正式 Participant，指导老师作为 Advisor（不计入队员，不参与线上评测与提交操作）。下面是基于比赛特点给出的分工与时间建议，仅供内部参考。

### 9.1 先到手的 Checklist（一周内完成）

1. **立即确认参赛资格**（Discord / 微信群），特别是 3 人学生队是否满足最少人数、Advisor 如何在表单中署名。
2. 在 **4 月 23 日之前** 完成注册（队长填表 → 所有学生队员收验证码 → 签 Informed Consent）。
3. 指定其中一位学生为 **Team Leader / 提交人**，统一负责每日提交和邮件沟通。
4. 拉下 **Starter Kit** 跑通 ReAct 基线，确保能在 `/input` → `/output` 接口上产出合格的 prediction.csv。
5. 下载 Phase 1 演示数据集，本地出一个 **baseline 分数**。
6. 检查自己集群能否模拟"16 核 / 64 GB / 无 GPU / 12 小时"的评测环境。

### 9.2 3 位学生的角色分工

| 角色 | 主要职责 |
|------|----------|
| **S1 · Agent 架构 + Planning（建议担任 Team Leader）** | 任务拆解、工具调用路由、ReAct / Plan-and-Execute 框架搭建；负责 Docker 打包与每日提交；统一对外沟通窗口 |
| **S2 · 数据与检索** | 异构数据加载（CSV / DB / 长文档 / Phase 2 图像视频）；长文档分块与检索（RAG）；列匹配评分器的本地复现与线下评测集维护 |
| **S3 · 提示工程 + 错误分析** | 针对 Qwen3.5-35B-A3B 做 prompt tuning；维护 bad-case 库，驱动下一轮迭代；Phase 2 承担多模态 prompt 设计 |

### 9.3 Advisor（指导老师）的职责范围

- 把控整体技术路线、实验设计、相关工作调研
- 审阅学生的方案文档与 Technical Report 草稿
- 不参与代码提交、不代学生答辩或沟通
- 在 Creative 子赛道和 KDD Cup Workshop 展示环节负责学术包装与论文撰写指导

### 9.3 技术风险提示

1. **评测 LLM 不可替换**：所有依赖 GPT-4 / Claude 的方案在正式评测阶段都会失效。设计 Agent 时就要 **以 Qwen3.5-35B-A3B 为唯一推理引擎** 做规划与实验。
2. **A3B 是多模态模型**：Phase 2 的图像/视频可以直接端到端调用；**不要**再搭外挂 VLM 当主推理引擎（违赛规）。
3. **Thinking 模式会吃输出 token**：简单子任务记得 `enable_thinking=False`，并设 `max_output_tokens` 硬上限。
4. **无 GPU 且 12h 总预算**：嵌入/重排模型如果用大模型，会吃掉大量时间。建议优先用轻量化 embedding（bge-small / bge-m3 / e5-small）+ 结构化索引。
5. **每天只能提交 1 次，Phase 1 共 30 次**：必须建 **本地评测集**，不要靠线上刷榜来调参；开发期通过阿里云百炼 API (¥0.4 / ¥3.2 per M tokens) 跑本地评测。
6. **Docker + 无外网**：所有模型、依赖、字典要 **打进镜像**；评测环境不能 `pip install`、不能 `wget`。
